# PlanLoop 产品设计文档（v1 规格）

> 日期：2026-06-21 · 状态：v1 已定稿，开始实现
> 一句话定位：**免费、极简、循环任务一目了然、不制造打卡焦虑的手机计划工具（PWA）。**

## 1. 背景与差异化

任务管理 + 习惯打卡是成熟红海（TickTick/滴答清单、Habitica、Loop、HabitNow、Streaks…）。
核心功能均为市场标配，拼功能赢不了大厂。本产品的差异化锚点：

1. **循环任务是一等公民、且全部免费**（滴答清单的循环藏得深、部分要付费 —— 用户的真实痛点）。
2. **低压力、反 streak 焦虑**：不显示"连续 N 天别断"、断了不掉色不掉血；总完成数刻意藏在次级页面。
3. **定量任务 + 智能模板个性化**（v3）：用权威公式给出个性化目标，是大厂没认真做的小切口。

## 2. 技术形态

- **纯前端 PWA**，手机浏览器打开即用，可"添加到主屏幕"。
- **本地存储**（localStorage，schema 版本化），**无后端、无登录、零成本**。
- **数据层独立封装**，未来加云同步不返工；后续同套代码可用 Capacitor 打包上 App Store。
- 零构建依赖（原生 HTML/CSS/JS），降低运行与分发门槛。

## 3. 信息架构（底部三 Tab）

- **今天**：默认页。列出今天该做的任务，点圆圈打卡。顶部轻量"本周完成 N"。已完成的周期任务折叠到「已完成」区。（v2：可切换日历视图）
- **计划**：所有任务的增删改查、设置类型与循环规则、查看单任务本周进度。
- **我的**：刻意藏起来的次级页。累计完成总数、本周概览、导出/导入备份、手动清理旧记录。（v3：个人资料入口）

## 4. 数据模型（一次设计到位，覆盖 v1–v3）

```
state = {
  schemaVersion: 1,
  profile: {},                       // v3 预留：gender/weight/age/activity（全部可选）
  settings: { weekStartsOn: 1 },     // 1 = 周一
  lifetimeTotal: 0,                  // 累计"已完成任务-天"净数；只随完成/撤销增减，清理旧记录不影响
  tasks: [Task],
  completions: [Completion]          // 明细日志，可被手动清理
}

Task = {
  id, title,
  kind: 'qualitative' | 'quantitative',   // 定性 / 定量
  unit: string|null,                       // 定量单位：ml/公里/分钟/页…
  target: number|null,                     // 定量目标（每日或每次）
  direction: 'atLeast' | 'atMost',         // 越多越好 / 越少越好
  multiAdd: boolean,                       // 一天内是否可多次累加（如喝水）
  quickAdds: [{label, amount}] | null,     // 定量快捷增量按钮
  recurrence: {
    type: 'daily'|'weekdays'|'weeklyCount'|'monthlyCount'|'once',
    weekdays: [1..7],                       // type=weekdays
    count: number,                         // type=weeklyCount/monthlyCount，周期内需完成 N 次（默认 1）
    date: 'YYYY-MM-DD'                      // type=once
  },
  createdAt, archived: false
}

Completion = { id, taskId, date:'YYYY-MM-DD', value: number|null, ts }
```

## 5. 关键逻辑

### 5.1 任务"今天是否显示"
- `daily`：每天显示。
- `weekdays`：今天星期几 ∈ weekdays 才显示。
- `weeklyCount` N：本周已完成次数 < N 时，**每天都显示**；达到 N 后移入「已完成」折叠区；下周一重置。
- `monthlyCount` N：同上，按自然月，1 号重置。
- `once`：仅 date 当天显示；完成后归入已完成。

### 5.2 "某天是否算完成"`isDoneOnDate(task, date)`
- 定性：当天存在一条 completion。
- 定量 `atLeast`：当天 value 之和 ≥ target。
- 定量 `atMost`：当天有记录且 value 之和 ≤ target（越少越好，达标=没超）。

### 5.3 打卡
- 定性：点一下 = 完成（再点取消）。
- 定量非多次：弹输入框，默认填 target，可改成真实值（少做/超额都照实记），一天一条。
- 定量多次（multiAdd，如喝水）：快捷按钮各加一条 completion 累加；达 target 即"完成"。

### 5.4 统计
- **本周完成数**：本周（周一–周日）内 `isDoneOnDate` 为真的「任务-天」计数；每周一自然归零（派生，不持久化）。
- **单任务本周进度**：本周已完成次数 / 计划次数（每天→7、周一三五→3、weeklyCount→N）。
- **累计总数 lifetimeTotal**：完成一个「任务-天」+1，撤销 -1；**手动清理旧记录不改动它**（关键：历史包袱可清，努力总量永留）。

### 5.5 备份与清理
- **导出**：整个 state 序列化为 JSON 文件下载。**导入**：读文件覆盖。（兜底本地数据丢失风险）
- **手动清理**：删除早于所选月数的 completion；lifetimeTotal 不动。

## 6. 设计语言
- 扁平、留白、0.5px 细边、柔和绿色表示完成；无连续天数、无红色警告。
- UI 全中文，句子式大小写。

## 7. 路线图

- **v1（本次）**：三页面 + 列表打卡 + 定性/定量（单位/目标/方向）+ 四种循环（每天 / 周固定几天 / 周·月 N 次 / 一次性）+ 多次累加打卡与快捷按钮（含喝水预设，用通用默认目标）+ 本周完成数 + 单任务进度 + 累计数 + 导出/导入 + 手动清理 + PWA 可安装。
- **v2（已完成，2026-06-22）**：喝水快捷键收成单个「+」点开展开；**计划页改为月历**（完成度热力 + 点某天补打卡）；**时段屏蔽**（全局点选起止日期按理由：旅行/受伤/放假/自定义，颜色区分；单任务休息时段如经期休息）。被屏蔽日不出现、不计入达成率、不打断 N 次配额、不影响累计数。
- **v2.x（已完成，2026-06-22）**：任务详情可视化——点开任务看定量柱状图（含目标虚线）/ 定性月度坚持热力图 + 本周圆点（不显示连续天数）。
- **v3（已完成，2026-06-22）**：个人资料（性别/体重/年龄/运动量，可选，仅存本机）；智能模板库（喝水/睡眠/走路/跑步/冥想/少刷手机/读书）；喝水按 EFSA(2010) 与 30–35ml/kg 体重法个性化推荐每日目标，小灰字标注来源。单口量按生理均值固定（≈30ml），个性化只作用于每日目标。
- **以后**：云同步、好友/分享、提醒通知、上架 App Store。
